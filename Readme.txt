1. Описание файлов проекта:
/client - клиент передачи файла по UDP
/server - сервер передачи файла по UDP
/classes - основные классы обеспечивающие передачу файлов
/include - заголовки классов

2. Сборка проекта.
Запуск сборки:
make

3. Проверка работоспособности.
Команда:
make run

Отработка:
будет запущен сервер и 5 копий клиента, отправлющих параллельно пакеты входного файла.
Вывод сервера попадает в файл s.txt, а клиентов в c<X>.txt, где <X>- порядковый номер запуска клиента,
кроме того распределение памяти сервера в процессе работы попадает (команда ps) в файл mem.txt .

Все файлы будут сформированы в процессе работы сервера и клиентов, а в конце работы команды run - они будут выведены.
Вывод клиента содержит сформированный  им fileID под которым передается файл, далее выводятся номера первых 
20 отправляемых пакетов и результат проверки CRC.

Сервер принимая параллельно несколько файлов, в момен завершения получения очередного файла выводит об этом 
соообщение и - сразу за ним - список состояния (количество принятых пакетов) для каждого файла, который еще 
в состоянии передачи находится.
